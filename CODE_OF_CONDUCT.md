# Code of Conduct

OpenVox Studio contributors are expected to communicate professionally and respectfully.

Be constructive in technical disagreement, critique code rather than people, respect different experience levels, and avoid harassment, threats, discrimination or deliberate disruption.

Maintainers may edit, hide or remove abusive contributions and may restrict participation when behavior repeatedly harms the project or its contributors.

Security reports and private user data must not be exposed publicly without a legitimate reason and appropriate consent.
